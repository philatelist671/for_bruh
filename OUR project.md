# 🗺️ NetMapper: Автоматическое построение карты сети

Интерактивный веб-инструмент для автоматического опроса коммутаторов H3C по SSH, сбора данных о подключенных устройствах и визуализации топологии сети в реальном времени.

## 📖 О проекте

NetMapper решает классическую задачу сетевого администратора: "какое устройство куда подключено?". Вместо ручного обхода коммутаторов и парсинга вывода консоли, система автоматически:
1. Подключается к коммутаторам по SSH.
2. Выполняет команды `display mac-address`, `display arp`, `display lldp neighbor`, `display arp snooping vlan`, ``.
3. Парсит вывод с помощью регулярных выражений.
4. Сохраняет данные в PostgreSQL с механизмом UPSERT (гарантия отсутствия дубликатов).
5. Генерирует JSON с графом сети через сложный SQL CTE-запрос.
6. Отрисовывает интерактивную карту в браузере с помощью Cytoscape.js.

Выполните скрипт для создания самоподписанного сертификата (срок действия 10 лет):

```bash
bash before_docker_compose_up_ssl_generate.sh
```

2. Настройка переменных окружения

Создайте файл `.env` в корне проекта (если его нет):
```.env
POSTGRES_USER=netmap_user 
POSTGRES_PASSWORD=netmap_secure_password
POSTGRES_DB=netmap_db
NGINX_PORT=443
APP_ENV=production
```

3. Запуск системы

```bash
docker compose up -d --build
```

Перейдите по адресу ```https://<IP-адрес-сервера>```.
```⚠️ Важно: Браузер предупредит о самоподписанном сертификате ("Ваше подключение не защищено"). Это ожидаемое поведение. Нажмите "Дополнительно" → "Перейти на сайт (небезопасно)".```

## 🔍 Описание ключевых модулей (Для разработчиков)

#### ```app/main.py``` **(Оркестратор)** (РЕАЛИЗОВАНО)

Точка входа FastAPI-приложения. Содержит эндпоинты для управления коммутаторами и запуска сканирования.
Ключевая особенность: Использует ```asyncio.Lock``` для обеспечения идемпотентности. Если пользователь нажмет кнопку "Построить карту" 10 раз подряд, запустится только один процесс, остальные получат HTTP 409 Conflict.

#### ```app/ssh_collector.py``` **(Сборщик данных)** (РЕАЛИЗОВАНО)

Отвечает за низкоуровневое SSH-подключение.
Ключевая особенность: Содержит monkey-patch для библиотеки ```paramiko```, который принудительно включает устаревшие алгоритмы ```ssh-rsa```. Это критически важно для подключения к коммутаторам H3C со старыми прошивками, которые иначе отвергают соединение с ошибкой ```IncompatiblePeer```.

#### ```app/parsers.py``` **(Парсер вывода)** (НЕ РЕАЛИЗОВАНО)

Превращает сырой текст из консоли коммутатора в структурированные словари Python.
**Ключевая особенность**: Выполняет строгую нормализацию MAC-адресов. Формат H3C ```0000-0000-0001``` преобразуется в стандартный ```00:00:00:00:00:01```, который принимает тип данных ```MACADDR``` в PostgreSQL.

#### ```app/db_manager.py``` **(Работа с БД)** (НЕ РЕАЛИЗОВАНО)

Использует асинхронный драйвер ```asyncpg```.
**Ключевая особенность**: Использует конструкцию ```INSERT ... ON CONFLICT DO UPDATE``` (UPSERT) для всех таблиц. Это гарантирует отсутствие дубликатов и автоматическое обновление поля ```last_seen``` при каждом успешном опросе.

#### ```app/json_generator.py``` **(Генератор карты)** (РЕАЛИЗОВАНО)

Содержит большой SQL-запрос с использованием CTE (Common Table Expressions), который собирает все сущности, определяет "хабы" (порты с ≥2 устройствами) и формирует итоговый граф ```{nodes: [...], edges: [...]}```.


## 🔄 Полный цикл работы (Pipeline)
1. Пользователь нажимает ```🔄 Построить карту``` в браузере.
2. JS отправляет ```POST /api/scan```.
3. FastAPI захватывает ```asyncio.Lock``` и запускает фоновую задачу (```BackgroundTask```).
4. Параллельный опрос всех коммутаторов из БД через SSH (в пуле потоков).
5. Парсеры превращают сырой текст в **словари Python**.
6. Данные сохраняются в PostgreSQL через ```UPSERT``` (без дубликатов).
7. CTE-запрос собирает актуальные данные и сохраняет их в ```/app/static/data.json```.
8. JS получает уведомление об успехе, делает ```GET /static/data.json```.
9. ```Cytoscape.js``` отрисовывает интерактивный граф сети.
